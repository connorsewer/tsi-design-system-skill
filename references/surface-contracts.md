# Surface Contracts

Use this reference to decide what "done" means for each common surface.

## Landing Page

Must include:

- Official logo in the header.
- Light premium default composition.
- One strong hero message and one visual artifact.
- One proof or mechanism section.
- One generated or official image used with intent.
- Final CTA with correct brand treatment.

Avoid:

- Centered text-only hero.
- Generic 4-card feature grid.
- Dark-only command-center mood unless the brief asks for it.

## Mini Deck

Must include:

- Opening slide with official logo.
- At least one visual/mechanism slide.
- At least one proof or synthetic metric slide.
- Closing slide with clear CTA.
- If producing `.pptx`, start from `uploads/TSI_PPT_Widescreen_Poppins.pptx`.

HTML decks must use inline CSS and correct relative asset paths.

## Product Visual Or Component

Must include:

- A coherent operational mechanism.
- Synthetic illustrative data only.
- Reduced-motion behavior.
- TSI color anchors and Poppins hierarchy.
- Clear distinction between generated/support visuals and official branding.

## Dashboard Or Portal

Must include:

- Correct logo placement.
- Synthetic data caveat when examples show metrics or records.
- Sub-brand/product context when relevant.
- Clear status color semantics.
- At least one brand-owned or generated support visual when the screen introduces a workflow.

## Generated-Image Prompt Book

Must include:

- Prompt intent and surface.
- Visual mode.
- Palette/type/brand anchors.
- Composition and safe-area guidance.
- Negative constraints.
- Metadata fields for review status and source.

## Social Or Campaign Graphic

Must include:

- One main idea.
- Official logo or correct mark placement.
- Either one strong generated image or one official people/sub-brand asset, not a cluttered mix.
- Safe areas for platform cropping.

## Written Collateral

Must include:

- Executive-modern voice.
- Clear claim hierarchy.
- No unsupported performance claims.
- No exclamation points or emoji.
- Current tagline only where appropriate.
